# Building Design Tree for a Design Document

Applies to any design document — OpenSpec `design.md`, RFC, ADR, PRD, tech spec, or equivalent.


- problem statement and goals
- non-goals and constraints
- architecture and boundaries
- data model and state transitions
- interfaces and contracts
- error handling and failure modes
- rollout, migration, and compatibility
- testing and operational validation

For each node, record:

- current decision status: decided | tentative | missing
- assumptions
- dependent nodes
- risk level

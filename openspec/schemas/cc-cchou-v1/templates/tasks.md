# Principal

- Think deeply and critically to discover edge cases and hidden complexities.
- Prioritize:
    * Simplicity
    * Testability
    * Explicitness
- Apply:
    * **YAGNI**
    * **DRY**.
    * **SOLID**
      - Single Responsibility Principle — each module should have one clear reason to change.
      - Open/Closed Principle — prefer extension points over modifying stable core behavior.
      - Liskov Substitution Principle — abstractions should be safely replaceable by implementations.
      - Interface Segregation Principle — prefer small, focused interfaces over broad ones.
      - Dependency Inversion — depend on abstractions at boundaries, not concrete implementations.
